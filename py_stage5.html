Step 5: Learn OOP Concepts ‚Äì Classes and Inheritance

In this stage you learn the basics of Object‚ÄëOriented Programming (OOP) in Python: how to define classes, create objects, and use inheritance to reuse and organize code.

OOP is used in most medium and large Python projects (Django apps, games, tools), so getting comfortable with these ideas will help you understand real‚Äëworld code.


1Ô∏è‚É£ What is a class? What is an object?

A class is like a blueprint or design that describes what data and behavior something will have, and an object is a real thing created from that blueprint.

  • Class = plan/template (for example, ‚ÄúStudent‚Äù design).
  • Object = actual item based on that plan (for example, Asha, Ravi as Student objects).
  • Classes define attributes (data) and methods (functions).
# Simple Student class (blueprint)
class Student:
    def __init__(self, name, branch, year):
        self.name = name
        self.branch = branch
        self.year = year

    def introduce(self):
        print("Hi, I am", self.name, "from", self.branch, "year", self.year)

# creating objects (students) from the class
s1 = Student("Asha", "CSE", 2)
s2 = Student("Ravi", "ECE", 3)

s1.introduce()
s2.introduce()

__init__ is a special method that runs when you create an object, and the first parameter self refers to ‚Äúthis object itself‚Äù.

2Ô∏è‚É£ Understanding __init__ and self

__init__() is called the constructor and sets up the object‚Äôs initial data (attributes).

  • self.name, self.branch are attributes stored inside each object.
  • When you call Student("Asha", "CSE", 2), Python automatically passes the new object as self.
  • Other parameters (like name, branch) come from what you pass when you create the object.
class Laptop:
    def __init__(self, brand, ram_gb):
        self.brand = brand
        self.ram_gb = ram_gb

    def show_config(self):
        print("Brand:", self.brand, "| RAM:", self.ram_gb, "GB")

l1 = Laptop("Dell", 8)
l2 = Laptop("HP", 16)

l1.show_config()
l2.show_config()

Each object has its own values (for example, one laptop has 8 GB RAM, another has 16 GB), but they share the same class definition.

3Ô∏è‚É£ Methods ‚Äì functions inside a class

Methods are functions defined inside a class that operate on the object‚Äôs data using self. 

class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount
        print("Deposited", amount, "New balance:", self.balance)

    def withdraw(self, amount):
        if amount <= self.balance:
            self.balance -= amount
            print("Withdrawn", amount, "New balance:", self.balance)
        else:
            print("Insufficient balance")

acc = BankAccount("Kiran", 1000)
acc.deposit(500)
acc.withdraw(300)
acc.withdraw(1500)

This pattern‚Äîclass plus attributes plus methods‚Äîis how many apps model real things like accounts, tickets, orders, and users.

4Ô∏è‚É£ What is inheritance?

Inheritance lets one class (child) reuse code from another class (parent) so the child automatically gets the parent‚Äôs attributes and methods.

  • Parent / base class ‚Äì general version (for example, Person).
  • Child / derived class ‚Äì more specific version (for example, Student, Teacher).
  • Child can add new features or change (override) some behavior.
# Parent class
class Person:
    def __init__(self, name):
        self.name = name

    def introduce(self):
        print("Hi, I am", self.name)

# Child class inherits Person
class Student(Person):
    def __init__(self, name, branch):
        super().__init__(name)   # call Person.__init__
        self.branch = branch

    def show_branch(self):
        print(self.name, "studies", self.branch)

p = Person("Asha")
p.introduce()

s = Student("Ravi", "CSE")
s.introduce()       # from Person
s.show_branch()     # from Student

The super() call lets the child reuse the parent‚Äôs initialization logic instead of repeating the same code in multiple classes.

5Ô∏è‚É£ Real‚Äëlife example ‚Äì College people

Here is a small example that models people in a college using inheritance. 

class Person:
    def __init__(self, name, role):
        self.name = name
        self.role = role

    def info(self):
        print(self.name, "-", self.role)

class Student(Person):
    def __init__(self, name, branch, year):
        super().__init__(name, "Student")
        self.branch = branch
        self.year = year

    def info(self):
        # override parent method
        print(self.name, "- Student -", self.branch, "Year", self.year)

class Teacher(Person):
    def __init__(self, name, subject):
        super().__init__(name, "Teacher")
        self.subject = subject

    def info(self):
        print(self.name, "- Teacher of", self.subject)

people = [
    Student("Asha", "CSE", 2),
    Student("Ravi", "ECE", 3),
    Teacher("Meera", "Python"),
]

for p in people:
    p.info()

This is how you can build bigger systems: start with a base class and extend it with more specific classes for each role.

6Ô∏è‚É£ When to use classes in your own projects

You do not need classes for every small script, but they are very helpful when:

  • You have many similar items (students, products, books, accounts) with the same kind of data.
  • You want to group related functions and data together, for example, all code about ‚ÄúStudent‚Äù inside one class.
  • You are building a larger project (web app, game, tool) where organizing code matters.

7Ô∏è‚É£ Mini program ‚Äì simple library system

This small example shows how you could start a library app using classes. 

class Book:
    def __init__(self, title, author, copies):
        self.title = title
        self.author = author
        self.copies = copies

    def available(self):
        return self.copies > 0

    def issue(self):
        if self.available():
            self.copies -= 1
            print("Issued:", self.title, "| Remaining:", self.copies)
        else:
            print("No copies left for:", self.title)

class Library:
    def __init__(self):
        self.books = []

    def add_book(self, book):
        self.books.append(book)

    def show_books(self):
        for b in self.books:
            print("-", b.title, "by", b.author, "| Copies:", b.copies)

lib = Library()
lib.add_book(Book("Python Basics", "Author A", 3))
lib.add_book(Book("Data Science Intro", "Author B", 1))

lib.show_books()

# issue some books
lib.books[0].issue()
lib.books[0].issue()
lib.books[1].issue()
lib.books[1].issue()

This is not a full library system, but it shows how classes can represent real objects and how methods implement actions on those objects.
