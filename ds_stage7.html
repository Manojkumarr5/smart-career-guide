Step 7: Learn Feature Engineering

In this stage you learn feature engineering: creating, cleaning, and transforming input columns (‚Äúfeatures‚Äù) so that machine learning models can learn patterns more easily.

Good feature engineering includes handling missing values, encoding categories, scaling numeric values, and sometimes creating new features from existing ones.


1Ô∏è‚É£ What is a feature?

A feature is a column used by a model to make predictions, such as study hours, attendance percentage, or branch of a student.

  • Numeric features: numbers like age, salary, marks.
  • Categorical features: labels like branch, city, product category.

Feature engineering means choosing which columns to use and how to represent them so that algorithms can understand them better.

2Ô∏è‚É£ Handle missing values (quick recap)

Before any further transformations, make sure missing values are handled in a consistent, documented way. 

import pandas as pd

df = pd.read_csv("students_raw.csv")

# Example: fill missing study_hours with median
df["study_hours"] = df["study_hours"].fillna(df["study_hours"].median())

# Example: drop rows where target (marks) is missing
df = df.dropna(subset=["marks"])

Many tutorials treat handling missing values as the first feature‚Äëengineering step, because unreliable data leads to weak features.

3Ô∏è‚É£ Encode categorical variables

Models cannot directly use text labels like branch names; these must be converted into numbers using encoding techniques. 

# Example dataset
# columns: name, branch (CSE/ECE/MECH), marks

# One‚Äëhot encode 'branch'
df_encoded = pd.get_dummies(df, columns=["branch"], drop_first=True)

print(df_encoded.head())

One‚Äëhot encoding (such as get_dummies()) is a common beginner‚Äëfriendly method for turning small categorical features into numeric columns.

4Ô∏è‚É£ Scale numeric features

Many algorithms work better when numeric features are on a similar scale (for example 0‚Äì1 or with mean 0 and standard deviation 1). 

from sklearn.preprocessing import StandardScaler

numeric_cols = ["study_hours", "attendance", "marks"]
scaler = StandardScaler()

df_scaled = df.copy()
df_scaled[numeric_cols] = scaler.fit_transform(df_scaled[numeric_cols])

print(df_scaled.head())

Distance‚Äëbased models like k‚Äënearest neighbors and clustering, as well as many linear models, benefit strongly from proper scaling.

5Ô∏è‚É£ Create simple new features

Sometimes combining existing columns into a new one can give the model a clearer signal, such as ratios or total scores. 

# Example: efficiency score = marks per study hour
df["efficiency"] = df["marks"] / (df["study_hours"] + 1e-3)

# Example: attendance category
df["attendance_level"] = pd.cut(
    df["attendance"],
    bins=[0, 60, 80, 100],
    labels=["low", "medium", "high"]
)

Simple, domain‚Äëbased features such as ratios or bucketed levels often improve model performance more than very complex transformations.
