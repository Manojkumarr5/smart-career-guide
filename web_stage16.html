Step 16: Deploy Your App on Netlify or Vercel

You built a complete web application locally. In Stage 16 you will learn how to put it online so anyone can access it from a URL using platforms like Netlify and Vercel.

These services connect directly to your GitHub repo and can automatically build and deploy your site when you push new changes.

Make sure your project is in a Git repository and pushed to GitHub so deployment platforms can pull the code.

This flow deploys a static or SPA frontend from Git to Netlify.

  1. Go to https://app.netlify.com and sign in.
  2. Click ‚ÄúNew site from Git‚Äù.
  3. Choose your Git provider, for example GitHub, and authorize access.
  4. Select your repository such as smart-career-guide.
  5. Set:
    • Build command: leave empty for pure static HTML/CSS/JS.
    • Publish directory: project root or dist if you use a bundler.
  6. Click ‚ÄúDeploy site‚Äù and wait until the deploy finishes, then copy the live URL.
Tip: For basic static projects without build tools, no build command is required; Netlify can directly serve your files.

Vercel provides a similar Git‚Äëbased deployment flow and is especially popular for modern frontends.

  2. Click ‚ÄúAdd New‚Äù ‚Üí ‚ÄúProject‚Äù.
  3. Import the same GitHub repo such as smart-career-guide.

Whenever you git push to the connected branch, Netlify or Vercel can automatically rebuild and redeploy your site as a basic CI/CD pipeline.
