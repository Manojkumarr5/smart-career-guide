Step 10: Learn a Frontend Framework (React / Vue Basics)

So far you built everything with plain HTML, CSS, and JavaScript. Frontend frameworks like React and Vue help you build bigger apps more easily using components and reusable logic.

In this stage you will understand the core ideas: components, props, state, and how they help manage complex user interfaces.

As your project grows, handling many DOM updates manually becomes hard and error‚Äëprone. Frameworks provide patterns so you can think in terms of components, not raw DOM operations.

React and Vue are two popular options. Both let you build reusable UI blocks and manage data flow more cleanly.

2Ô∏è‚É£ Components: building blocks of UI

A component is a reusable piece of UI with its own structure, style, and logic.

In React, a basic functional component is a JavaScript function that returns JSX.

// React example (concept only)
function CourseCard(props) {
  return (
    <div className="card">
      <h3>{props.title}</h3>
      <p>{props.description}</p>
    </div>
  );
}

// Using the component
<CourseCard title="Web Development" description="Learn HTML, CSS, JS." />

JSX looks like HTML, but it is actually JavaScript syntax that React converts into DOM operations.

3Ô∏è‚É£ Props: passing data into components

Props (properties) are how you pass data from a parent component to a child component.

They are read‚Äëonly inside the child and make components reusable.

// Parent component using CourseCard multiple times
function CourseList() {
  return (
    <div>
      <CourseCard
        title="Machine Learning"
        description="Build intelligent systems."
      />
      <CourseCard
        title="Web Development"
        description="Create modern websites."
      />
    </div>
  );
}

In Vue, props work similarly: you define props in the child and pass values from the parent.

4Ô∏è‚É£ State: data that can change

State is data that belongs to a component and can change over time.

When state changes, the component re‚Äërenders to show the new values.

// React example using useState (concept)
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0); // state variable + setter

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}

State is local to the component; props are data coming from outside.

5Ô∏è‚É£ One‚Äëway data flow

React and Vue encourage one‚Äëway data flow: data moves from parent to child via props.

Children can notify parents using callback functions passed as props, making data flow easier to reason about.

// Parent manages state, passes down value and handler
function Parent() {
  const [selectedCourse, setSelectedCourse] = useState("None");

  return (
    <div>
      <p>Selected: {selectedCourse}</p>
      <CourseSelector
        onSelect={(name) => setSelectedCourse(name)}
      />
    </div>
  );
}

// Child calls parent's handler
function CourseSelector(props) {
  return (
    <div>
      <button onClick={() => props.onSelect("Web Development")}>
        Choose Web
      </button>
    </div>
  );
}
