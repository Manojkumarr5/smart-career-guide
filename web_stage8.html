Step 8: Use Git & GitHub for Version Control

From Stage 1 to 7 you built real web pages and mini projects. Now it is time to learn how to save your work safely, track changes, and share your code using Git and GitHub.

Git is a version control system on your computer. GitHub is a website where you can store your Git repositories online and show them to others.


1Ô∏è‚É£ Why version control?

Without version control, you may end up with many folders like project_final, project_final2, project_really_final.

With Git, every important change is saved as a commit with a message, and you can go back to older versions.

2Ô∏è‚É£ Basic Git and GitHub concepts

Some simple words you must know:

  • Repository (repo): A project folder that Git tracks.
  • Commit: A snapshot of your changes with a message.
  • Remote: A copy of your repo stored online (for example, on GitHub).
  • Push: Send commits from your computer to GitHub.
  • Pull: Get latest commits from GitHub to your computer.

3Ô∏è‚É£ One-time setup (install Git and configure)

Before using Git, install it and set your name and email (shown in commit history).

Open your terminal (Command Prompt, PowerShell, or Git Bash) and run:

# Set your name (shown in commits)
git config --global user.name "Your Name"

# Set your email (should match your GitHub email)
git config --global user.email "you@example.com"

4Ô∏è‚É£ Create a local Git repository

Tell Git to track your existing Smart Career Guide project folder.

# 1. Go to your project folder
cd path/to/your/project

# 2. Initialize git (create .git folder)
git init

# 3. See current status
git status

After git init, Git starts watching your files but nothing is committed yet.

5Ô∏è‚É£ Add and commit files

Add files to the staging area, then commit them with a message.

# Add all files to staging area
git add .

# Create first commit
git commit -m "Initial commit - Smart Career Guide project"

# Check status again
git status

Now your local repository has one commit with all your current files.

6Ô∏è‚É£ Create a GitHub repository

Next, create a remote repo on GitHub to store your code online.

  • Log in and click the + New repository button.
  • Give it a name, for example smart-career-guide.
  • Keep it Public (or Private if you prefer).
  • Click Create repository.

GitHub will show you a URL like https://github.com/yourname/smart-career-guide.git.

7Ô∏è‚É£ Connect local repo to GitHub and push

Connect your local repo to the remote and push your commits.

# 1. Add the remote (replace URL with your repo URL)
git remote add origin https://github.com/yourname/smart-career-guide.git

# 2. Push commits from local 'main' branch to GitHub
git branch -M main
git push -u origin main

Refresh your GitHub repo page. You should see all your project files.

8Ô∏è‚É£ Daily workflow: change, add, commit, push

Whenever you work on your project, repeat this routine:

# See what changed
git status

# Stage changed files
git add .

# Commit with a useful message
git commit -m "Add web_stage7 and web_stage8 pages"

# Push to GitHub
git push

On another day, before starting work, run git pull to get latest changes if you use multiple devices or teammates.
