Step 13: Learn Model Tuning and Evaluation

In this stage you learn how to properly evaluate a model and improve it using hyperparameter tuning (changing model settings like depth or learning rate).

Instead of only checking accuracy once, you use better metrics and tools like cross‚Äëvalidation and GridSearchCV to find stronger models in a systematic way.


1Ô∏è‚É£ Evaluation metrics for classification and regression

Different problems need different metrics: classification uses accuracy, precision, recall, F1; regression uses MAE, MSE, RMSE, and R¬≤.

  • Accuracy: percent of correct predictions, but can be misleading on imbalanced data.[web:447][web:450]
  • Precision and recall: focus on the quality of positive predictions, very important for minority classes.[web:445][web:450]
  • MSE/RMSE: average squared error and its square root; R¬≤ shows how much variance is explained for regression.[web:447]
from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    mean_squared_error, r2_score
)

# Example for classification
y_true = [0, 0, 1, 1, 1]
y_pred = [0, 0, 0, 1, 1]

print("Accuracy :", accuracy_score(y_true, y_pred))
print("Precision:", precision_score(y_true, y_pred))
print("Recall   :", recall_score(y_true, y_pred))
print("F1-score :", f1_score(y_true, y_pred))

Beginner tutorials show how different metrics highlight different model behaviors, especially when classes are imbalanced.

2Ô∏è‚É£ Cross‚Äëvalidation ‚Äì more reliable evaluation

Cross‚Äëvalidation (such as k‚Äëfold) splits the training data into several folds so the model is trained and tested multiple times on different parts.

  • This gives a more stable performance estimate than a single train‚Äëtest split.
  • Common choice: 5‚Äëfold or 10‚Äëfold cross‚Äëvalidation.
from sklearn.model_selection import cross_val_score
from sklearn.linear_model import LogisticRegression

model = LogisticRegression(max_iter=1000)

scores = cross_val_score(
    model, X, y,
    cv=5,              # 5-fold cross-validation
    scoring="accuracy"
)
print("CV scores:", scores)
print("Mean accuracy:", scores.mean())

Model selection guides use cross‚Äëvalidation to reduce variance in evaluation and compare different models fairly.

3Ô∏è‚É£ What is hyperparameter tuning?

Hyperparameters are model settings chosen before training (like tree depth or number of neighbors); tuning means searching for values that give better performance.

  • Examples: n_neighbors in k‚ÄëNN, max_depth in decision trees, C in logistic regression.[web:449]
  • Automatic search: GridSearchCV and RandomizedSearchCV in scikit‚Äëlearn.

Best practices suggest tuning on training data using cross‚Äëvalidation, then checking the final chosen model once on a held‚Äëout test set.

4Ô∏è‚É£ Simple GridSearchCV example

GridSearchCV tries many combinations of hyperparameters and uses cross‚Äëvalidation to find the best one. 

from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(random_state=42)

param_grid = {
    "n_estimators": [50, 100],
    "max_depth": [None, 5, 10]
}

grid_search = GridSearchCV(
    estimator=rf,
    param_grid=param_grid,
    cv=3,
    scoring="accuracy"
)

grid_search.fit(X_train, y_train)

print("Best params :", grid_search.best_params_)
print("Best score  :", grid_search.best_score_)

best_model = grid_search.best_estimator_
y_pred = best_model.predict(X_test)
print("Test accuracy:", accuracy_score(y_test, y_pred))

GridSearchCV integrates cross‚Äëvalidation directly into the hyperparameter search, reducing the need for manual loops and helping avoid overfitting to one split.
