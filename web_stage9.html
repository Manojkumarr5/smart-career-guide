Step 9: Understand Responsive Design Principles

In Stage 3 you used Flexbox and Grid to build responsive layouts. In Stage 9 you will go deeper into the principles behind responsive design, so your pages work well on all devices.

You will learn about mobile‚Äëfirst thinking, breakpoints, fluid layouts, responsive images, and typography.


1Ô∏è‚É£ What is responsive design?

Responsive design means your website adapts to different screen sizes and devices automatically.

The same HTML works on phones, tablets, and desktops, with layout changes controlled by CSS.

2Ô∏è‚É£ Mobile-first approach

Mobile‚Äëfirst means designing and coding for the smallest screens first, then enhancing the layout for larger screens.

In CSS, mobile‚Äëfirst usually means writing base styles for mobile and then using @media (min-width: ...) queries to adjust for tablets and desktops.

/* Base styles: mobile first */
body {
  font-size: 16px;
}

.card-grid {
  display: grid;
  grid-template-columns: 1fr; /* 1 column on small screens */
  gap: 10px;
}

/* Tablet and up */
@media (min-width: 600px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Desktop and up */
@media (min-width: 992px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);
  }
}

3Ô∏è‚É£ Let content determine breakpoints

Instead of only using common device widths, resize your layout and add breakpoints where the design starts to break.

Breakpoints should follow content needs, not just device names.

4Ô∏è‚É£ Example: responsive two-column layout

This example shows a header, sidebar, main content, and footer that switch from two columns to stacked on small screens.

/* Layout container */
.demo-layout {
  border: 1px solid #dde6ff;
  padding: 10px;
  border-radius: 8px;
}

/* Main area using Flexbox */
.demo-main {
  display: flex;
  gap: 10px;
}

/* Sidebar and content */
.demo-sidebar {
  flex: 1;
}

.demo-content {
  flex: 2;
}

/* On small screens, stack vertically */
@media (max-width: 768px) {
  .demo-main {
    flex-direction: column;
  }
}

5Ô∏è‚É£ Responsive typography

Use relative units like rem and em and adjust sizes slightly at larger breakpoints for comfortable reading.

/* Base font sizing */
html {
  font-size: 16px; /* 1rem = 16px */
}

h1 {
  font-size: 1.8rem;
}

p {
  font-size: 1rem;
  line-height: 1.6;
}

/* Larger screens: slightly bigger text */
@media (min-width: 992px) {
  h1 {
    font-size: 2.2rem;
  }
  p {
    font-size: 1.05rem;
  }
}

6Ô∏è‚É£ Responsive images

Make images scale down using max-width: 100% and height: auto so they do not overflow the container.

img {
  max-width: 100%;
  height: auto;
  display: block;
}
