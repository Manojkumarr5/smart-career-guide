Step 3: Work with Data Types, Lists, and Dictionaries

In this stage you learn how Python stores information using different data types, and how to organize many values using lists and dictionaries. These are the basic containers you will use again and again in data science, web apps, and automation.

By the end of this step, you should feel comfortable creating lists and dictionaries, reading values from them, and updating them in simple programs.


1Ô∏è‚É£ Core Python data types

Python has several built‚Äëin data types. For now, focus on the most common ones.

  • int ‚Äì whole numbers like 5, -3, 2025.
  • float ‚Äì decimal numbers like 3.14, 0.5.
  • str ‚Äì text (strings) like "hello", "Smart Career Guide".
  • bool ‚Äì logical values: True or False.
  • list ‚Äì ordered, changeable collection of items.
  • dict ‚Äì collection of key‚Äìvalue pairs (like a small database row).
age = 19              # int
price = 199.99        # float
name = "Ananya"       # str
is_student = True     # bool

You can check a value‚Äôs type using the type() function, for example: type(age), type(name).

2Ô∏è‚É£ Lists ‚Äì ordered collections

A list is like a container that can hold many values in order. You create lists with square brackets []. Lists can store numbers, strings, or even a mix. 

marks = [78, 85, 90, 66]
friends = ["Asha", "Ravi", "Kiran"]
mixed = ["Ravi", 19, True]

Accessing items (indexing starts from 0):

print(friends[0])   # "Asha"
print(marks[2])     # 90

# negative index from the end
print(friends[-1])  # "Kiran"

Changing and adding items:

marks[1] = 88               # change second mark
marks.append(95)            # add at the end
friends.insert(1, "Meera")  # insert at position 1

print(marks)
print(friends)

Lists are mutable (changeable): you can update, add, and remove items after creating them.

3Ô∏è‚É£ Looping over lists

Lists become powerful when you combine them with loops. You can easily process all items. 

marks = [78, 85, 90, 66]

total = 0
for m in marks:
    total = total + m

average = total / len(marks)
print("Average mark:", average)

This simple pattern (list + loop + calculation) is used in many real tasks: attendance, marks, sales, sensor readings, and more.

4Ô∏è‚É£ Dictionaries ‚Äì key‚Äìvalue pairs

A dictionary stores data as key ‚Üí value pairs. You create dictionaries with curly braces {}. It is like a small database row where you can quickly look up a value using its key. 

student = {
    "name": "Asha",
    "age": 19,
    "branch": "CSE",
    "is_hostel": True
}

print(student["name"])    # "Asha"
print(student["branch"])  # "CSE"

Adding and updating values:

student["cgpa"] = 8.5          # add new key
student["age"] = 20            # update age

print(student)

Keys are usually strings (like "name" or "cgpa"). Dictionaries are very common in APIs, JSON, and config files.[web:40][web:43]

5Ô∏è‚É£ List of dictionaries ‚Äì real student data

You can combine lists and dictionaries to store many records, for example a list of students where each student is a dictionary.[web:42][web:47] 

students = [
    {"name": "Asha", "branch": "CSE", "cgpa": 8.5},
    {"name": "Ravi", "branch": "ECE", "cgpa": 7.8},
    {"name": "Kiran", "branch": "ME", "cgpa": 9.1},
]

for s in students:
    print(s["name"], "-", s["branch"], "-", s["cgpa"])

This pattern is very close to real applications, where data is stored as a list of records (students, products, employees, etc.).

6Ô∏è‚É£ Small comparison ‚Äì list vs dictionary

Both lists and dictionaries can hold many values, but they are used differently.

  • Use a list when order matters and you access items by position (0, 1, 2, ‚Ä¶), for example marks, daily temperatures, or a list of names.
  • Use a dictionary when you want to label each value with a key and access by name, for example one student‚Äôs profile or one product‚Äôs details.

7Ô∏è‚É£ Simple mini program ‚Äì marks report

Here is a small example using everything from this stage: data types, list, dict, and loops. 

students = [
    {"name": "Asha", "marks": [78, 85, 90]},
    {"name": "Ravi", "marks": [60, 55, 70]},
    {"name": "Kiran", "marks": [95, 92, 88]},
]

for s in students:
    total = sum(s["marks"])
    avg = total / len(s["marks"])

    if avg >= 75:
        status = "Distinction"
    elif avg >= 35:
        status = "Pass"
    else:
        status = "Fail"

    print(s["name"], ":", avg, "-", status)

This is already close to a real college mini‚Äëproject: basic result processing using Python data types instead of Excel.
