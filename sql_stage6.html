Step 6: Master JOINs (INNER, LEFT, RIGHT, FULL)

In this stage you learn how to combine data from multiple tables using SQL JOINs: INNER JOIN, LEFT JOIN, RIGHT JOIN, and FULL JOIN.

These joins let you answer questions that need information spread across tables, such as students and their marks or customers and their orders.


1Ô∏è‚É£ Setup: students and marks tables

Use two related tables for practice: one for students and one for their marks, connected by a foreign key such as student_id. 

-- students table
CREATE TABLE students (
    id      INT PRIMARY KEY,
    name    VARCHAR(100),
    branch  VARCHAR(50)
);

-- marks table
CREATE TABLE marks (
    id         INT PRIMARY KEY,
    student_id INT,
    subject    VARCHAR(50),
    score      INT
);

Insert rows so that some students have marks in multiple subjects and at least one student has no marks; this makes join differences clear.

2Ô∏è‚É£ INNER JOIN ‚Äì only matching rows

INNER JOIN returns only rows where the join condition matches in both tables, usually linking a primary key to a foreign key. 

-- list students with their subjects and scores
SELECT s.id,
       s.name,
       s.branch,
       m.subject,
       m.score
FROM students AS s
INNER JOIN marks AS m
    ON s.id = m.student_id;

Any student without a matching row in marks will not appear because INNER JOIN keeps only matching pairs.

3Ô∏è‚É£ LEFT JOIN ‚Äì keep all left table rows

LEFT JOIN returns all rows from the left table and matching rows from the right table, filling non‚Äëmatches with NULL values. 

-- list all students and show marks if they exist
SELECT s.id,
       s.name,
       s.branch,
       m.subject,
       m.score
FROM students AS s
LEFT JOIN marks AS m
    ON s.id = m.student_id;

Students with no marks still appear, with NULL for subject and score, which helps identify missing data.

4Ô∏è‚É£ RIGHT JOIN and FULL JOIN ‚Äì idea and syntax

RIGHT JOIN keeps all rows from the right table and matching rows from the left, while FULL JOIN keeps all rows from both tables and uses NULLs where no match exists; some databases do not support FULL JOIN directly. 

-- RIGHT JOIN pattern (supported in many but not all SQL systems)
SELECT s.id,
       s.name,
       s.branch,
       m.subject,
       m.score
FROM students AS s
RIGHT JOIN marks AS m
    ON s.id = m.student_id;

-- FULL JOIN pattern (for systems that support FULL OUTER JOIN)
SELECT s.id,
       s.name,
       s.branch,
       m.subject,
       m.score
FROM students AS s
FULL JOIN marks AS m
    ON s.id = m.student_id;

INNER JOIN and LEFT JOIN cover most everyday needs; RIGHT and FULL joins are used more selectively and may require workarounds in some databases.

5Ô∏è‚É£ JOINs with filtering and sorting

You can combine joins with WHERE and ORDER BY to answer focused questions, such as filtering by branch and ordering by score. 

-- CSE students only, sorted by score descending
SELECT s.name,
       s.branch,
       m.subject,
       m.score
FROM students AS s
INNER JOIN marks AS m
    ON s.id = m.student_id
WHERE s.branch = 'CSE'
ORDER BY m.score DESC;

A helpful mental template is: FROM table1 JOIN table2 ON join_condition WHERE ... ORDER BY ... and then fill each part.
