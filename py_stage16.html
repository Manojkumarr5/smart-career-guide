Step 16: Deploy Small Web Apps

In this final Python stage you learn the basic idea of deploying your Flask web app to the internet using modern platforms such as Render or Railway.

The goal is not to become a DevOps expert but to understand how to prepare your app, create requirements.txt, and push code so others can open a URL and see it.

Start with a simple Flask app that you already tested locally, for example from Stage 13 or 15.

app = Flask(__name__)

@app.route("/")
def home():
    return "Hello from Smart Career Guide deployment!"

    app.run()

Keep the app small at first; once deployment works you can plug in your bigger project such as an API, dashboard, or student portal.

3Ô∏è‚É£ Add Procfile (or start command)

Many platforms look for a file that tells them how to start your app, often a Procfile with a single line.

web: gunicorn app:app

This line says: run the ‚Äúweb‚Äù process using gunicorn, pointing to app.py and the Flask object app.

4Ô∏è‚É£ Put project under Git and push to GitHub

Platforms like Render or Railway often connect directly to a GitHub repository. 

# inside your project folder
git init
git add .
git commit -m "Initial deployable version"

# then create a GitHub repo and push (replace URL)
git remote add origin https://github.com/yourusername/smart-career-flask.git
git branch -M main
git push -u origin main

Having your app on GitHub also helps build a portfolio and enables easy code reviews.

5Ô∏è‚É£ Deploy to a platform (example: Railway / Render)

Deployment screens differ slightly between platforms, but the overall flow is similar.

  • Create an account on a host such as Railway or Render.
  • Click ‚ÄúNew Project‚Äù or ‚ÄúNew Web Service‚Äù and choose ‚ÄúDeploy from GitHub‚Äù.
  • Select your Flask repository and confirm defaults for a Python app and main branch.
  • The platform installs dependencies from requirements.txt and runs your start command.

After a successful build, the platform gives you a public URL where your app is live.

6Ô∏è‚É£ Basic environment variables idea

In real apps you should not hardcode secrets such as API keys or database passwords and instead keep them as environment variables configured on the hosting platform.

  • In Flask, read values using os.environ.get("SECRET_KEY") after importing os.
  • On Railway or Render, open project settings, find ‚ÄúEnvironment Variables‚Äù, and add key‚Äìvalue pairs.

This pattern is used in almost all serious web apps for security and flexibility.
