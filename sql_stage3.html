Step 3: Learn CREATE, DROP, and ALTER TABLE

In this stage you learn the main DDL (Data Definition Language) commands: CREATE TABLE, ALTER TABLE, and DROP TABLE, which control the structure of your tables.

You will create new tables, change existing tables by adding or removing columns, and safely drop test tables you no longer need.


1Ô∏è‚É£ CREATE TABLE ‚Äì make a new table

CREATE TABLE defines a new table with its columns and data types. 

CREATE TABLE students (
    id      INTEGER PRIMARY KEY,
    name    VARCHAR(100) NOT NULL,
    branch  VARCHAR(50),
    cgpa    DECIMAL(3,1)
);
  • id INTEGER PRIMARY KEY ‚Äì unique id for each student.
  • name VARCHAR(100) NOT NULL ‚Äì text up to 100 characters, cannot be empty.
  • cgpa DECIMAL(3,1) ‚Äì numeric value like 8.5 or 9.0.

In your tool (MySQL, PostgreSQL, or SQLite GUI), run this command in the SQL editor connected to your practice database.

2Ô∏è‚É£ CREATE another table ‚Äì example marks table

Here is a second table that can store exam scores for students. 

CREATE TABLE marks (
    id         INTEGER PRIMARY KEY,
    student_id INTEGER,
    subject    VARCHAR(50),
    score      INTEGER
);

Later you will connect student_id to students.id with a foreign key; here the focus is just on table structure.

3Ô∏è‚É£ ALTER TABLE ‚Äì add a new column

ALTER TABLE changes the structure of an existing table, such as adding, modifying, or dropping columns. 

-- Add a new column 'email' to students
ALTER TABLE students
ADD COLUMN email VARCHAR(100);

After this, the students table has an extra column at the end; existing rows get NULL for email until updated.

4Ô∏è‚É£ ALTER TABLE ‚Äì rename or change column (concept)

Different database systems have slightly different syntax for renaming or changing column types, but the idea is to ALTER the column definition. 

-- Example pattern (exact syntax may differ per DB)
-- Rename column 'branch' to 'department'
-- Example (PostgreSQL-style idea):
-- ALTER TABLE students RENAME COLUMN branch TO department;

-- Change data type idea:
-- ALTER TABLE students ALTER COLUMN cgpa TYPE DECIMAL(4,2);

At beginner level, focus on adding and dropping columns; renaming and type changes are less frequent but useful to recognize.

5Ô∏è‚É£ DROP TABLE ‚Äì delete a whole table

DROP TABLE removes a table and all of its data from the database, so use it carefully. 

-- Remove the marks table completely
DROP TABLE marks;

You usually drop tables only in test databases, during development, or when cleaning structures you no longer need.

6Ô∏è‚É£ DDL vs DML ‚Äì quick distinction

CREATE, ALTER, and DROP are DDL (Data Definition Language) because they define or change the structure of tables.

  • DDL: CREATE TABLE, ALTER TABLE, DROP TABLE, TRUNCATE, RENAME, etc.
  • DML: SELECT, INSERT, UPDATE, DELETE ‚Äì used to work with the actual data rows.
