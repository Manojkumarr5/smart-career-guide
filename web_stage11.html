Step 11: Learn Backend (Node.js & Express Basics)

Until now, you worked mainly on the frontend: HTML, CSS, JavaScript in the browser, and calling public APIs. Backend development is about what happens on the server: handling requests, processing data, talking to databases, and sending responses back.

In this stage you will learn what backend is and build a simple Node.js + Express server with a few routes.


1Ô∏è‚É£ What is backend development?

Backend (server‚Äëside) development includes the logic, APIs, and databases that power your application behind the scenes.

The frontend sends requests, and the backend responds with data or actions such as reading or updating records.

2Ô∏è‚É£ What is Node.js and Express?

Node.js lets you run JavaScript on the server instead of only in the browser.

Express is a lightweight Node.js framework that makes it easier to define routes like app.get() and app.post().

3Ô∏è‚É£ Setup: create a simple Express server

Follow these steps to start a basic backend project.

# 1. Create a folder and go inside it
mkdir scg-backend
cd scg-backend

# 2. Initialize Node.js project (creates package.json)
npm init -y

# 3. Install Express
npm install express

Now create a file called server.js and put this basic code inside.

// server.js
const express = require("express");
const app = express();

// Middleware to parse JSON bodies
app.use(express.json());

// Simple home route
app.get("/", (req, res) => {
  res.send("Smart Career Guide Backend is running!");
});

// Start the server
const PORT = 3000;
app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});

Run your server with:

node server.js

Open http://localhost:3000 in your browser to see the home route message.

4Ô∏è‚É£ Creating simple API routes (GET)

Express lets you define routes for different URLs, each with a handler that receives a request and sends a response.

Add these routes in server.js to serve sample course data.

// Fake in-memory data
const courses = [
  { id: 1, name: "Web Development", level: "Beginner" },
  { id: 2, name: "Machine Learning", level: "Intermediate" },
  { id: 3, name: "Data Science", level: "Intermediate" }
];

// Get all courses
app.get("/api/courses", (req, res) => {
  res.json(courses);
});

// Get one course by id
app.get("/api/courses/:id", (req, res) => {
  const id = Number(req.params.id);
  const course = courses.find(c => c.id === id);

  if (!course) {
    return res.status(404).json({ error: "Course not found" });
  }

  res.json(course);
});

Visit http://localhost:3000/api/courses for all courses and /api/courses/1 for a single course.

5Ô∏è‚É£ Handling POST requests (create new course)

Use POST to create new data by reading JSON from the request body and pushing it into your in‚Äëmemory array.

// Create a new course (POST)
app.post("/api/courses", (req, res) => {
  const { name, level } = req.body;

  if (!name || !level) {
    return res.status(400).json({ error: "Name and level are required" });
  }

  const newCourse = {
    id: courses.length + 1,
    name,
    level
  };

  courses.push(newCourse);

  res.status(201).json(newCourse);
});

Test POST routes using tools like Postman, Thunder Client, or a frontend fetch call.

6Ô∏è‚É£ Connecting frontend and backend (concept)

Your backend is just another API that your frontend can call using fetch, similar to public APIs from Stage 7.

Later you can load Smart Career Guide course data from http://localhost:3000/api/courses instead of hard‚Äëcoding arrays.

// Example: Fetch courses from your backend
fetch("http://localhost:3000/api/courses")
  .then(res => res.json())
  .then(data => {
    console.log("Courses from backend:", data);
  })
  .catch(err => {
    console.error("Error:", err);
  });
