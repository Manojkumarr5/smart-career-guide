Step 17: Practice SQL Interview Questions

In this stage you convert your SQL knowledge into interview strength by solving common coding and theory questions on your own tables.

Focus on recognizing patterns such as filters, joins, aggregates, and subqueries instead of memorizing fixed answers.


1Ô∏è‚É£ Basic query and filtering questions

Interviews usually start with SELECT and WHERE questions that test your comfort with basic row filtering. 

-- Q1: list all CSE students with cgpa >= 8.0
SELECT id, name, branch, cgpa
FROM students
WHERE branch = 'CSE'
  AND cgpa >= 8.0;

-- Q2: fetch students whose name starts with 'A'
SELECT id, name
FROM students
WHERE name LIKE 'A%';

-- Q3: show students who are not from CSE or ECE
SELECT id, name, branch
FROM students
WHERE branch NOT IN ('CSE', 'ECE');

Similar questions appear for employees, products, or orders tables using WHERE, IN/NOT IN, BETWEEN, and LIKE.

2Ô∏è‚É£ JOIN and relationship questions

Next, interviewers check whether you can combine related tables using joins to answer business-style questions. 

-- Q4: list students who have at least one marks record
SELECT DISTINCT s.id, s.name
FROM students AS s
JOIN marks    AS m
  ON s.id = m.student_id;

-- Q5: list students who do NOT have any marks record
SELECT s.id, s.name
FROM students AS s
LEFT JOIN marks AS m
  ON s.id = m.student_id
WHERE m.student_id IS NULL;

-- Q6: fetch student names with their total marks
SELECT s.name,
       SUM(m.score) AS total_score
FROM students AS s
JOIN marks    AS m
  ON s.id = m.student_id
GROUP BY s.name;

Patterns like ‚Äúin A but not in B‚Äù or ‚Äújoin then aggregate per group‚Äù show up frequently in analytics and engineering interviews.

3Ô∏è‚É£ Aggregation, GROUP BY, and HAVING

Many questions use COUNT, SUM, AVG, GROUP BY, and HAVING to check your ability to summarize and filter groups. 

-- Q7: count students in each branch
SELECT branch,
       COUNT(*) AS total_students
FROM students
GROUP BY branch;

-- Q8: branches with average cgpa >= 8.0
SELECT branch,
       AVG(cgpa) AS avg_cgpa
FROM students
GROUP BY branch
HAVING AVG(cgpa) >= 8.0;

-- Q9: find duplicate emails in users table
SELECT email,
       COUNT(*) AS cnt
FROM users
GROUP BY email
HAVING COUNT(*) > 1;

Variations include ‚Äútop N by some metric‚Äù, ‚Äúfind duplicates‚Äù, and ‚Äúfilter groups using HAVING after GROUP BY‚Äù.

4Ô∏è‚É£ Subquery and top‚ÄëN questions

Subqueries appear in tasks like ‚Äúabove average‚Äù, ‚Äúnth highest value‚Äù, or comparing against aggregated results. 

-- Q10: students with cgpa above overall average
SELECT name, branch, cgpa
FROM students
WHERE cgpa > (
    SELECT AVG(cgpa)
    FROM students
);

-- Q11: top 3 students by total score
SELECT name, total_score
FROM (
    SELECT s.name,
           SUM(m.score) AS total_score
    FROM students AS s
    JOIN marks    AS m
      ON s.id = m.student_id
    GROUP BY s.name
) AS t
ORDER BY total_score DESC
LIMIT 3;

The same idea is reused for employees with above-average salary or customers with top revenue.

5Ô∏è‚É£ Theory and scenario-style questions

You will also face short theory and scenario questions to test core concepts and communication skills.

  • Explain the difference between WHERE and HAVING with a simple grouped query.
  • Explain INNER JOIN vs LEFT JOIN using a students/marks example.
  • Explain DELETE, TRUNCATE, and DROP at a high level: row-level delete, table reset, and structure removal.
  • Discuss how to speed up a slow query: indexes, avoiding SELECT *, better filters, and reviewing execution plans.

Practice answering aloud in simple language; interviewers care about clear explanations as much as correct SQL.
