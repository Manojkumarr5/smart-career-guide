Step 7: Learn APIs and Fetch Calls

In Stage 6, you built small projects fully on the front‚Äëend. In Stage 7, you learn how your web app can talk to servers using APIs and the Fetch API.

This is how websites get live data such as weather, user profiles, posts, and more, from other services.


1Ô∏è‚É£ What is an API (simple explanation)?

API stands for Application Programming Interface. In web development, it usually means a set of URLs that you can call to send or get data from a server.

Think of an API like a waiter in a restaurant: you ask for something, the API talks to the server, and brings back the result as data.

2Ô∏è‚É£ What is the Fetch API?

The Fetch API is a built‚Äëin JavaScript feature that lets you make HTTP requests (GET, POST, etc.) to URLs and process the responses.

Basic pattern: call fetch(url), wait for the response, convert it to JSON with response.json(), then use the data.

// Basic Fetch example (GET request)
fetch("https://jsonplaceholder.typicode.com/users")
  .then(response => response.json())
  .then(data => {
    console.log("Users:", data);
  })
  .catch(error => {
    console.error("Error fetching users:", error);
  });

3Ô∏è‚É£ Using Fetch to get JSON data

Many APIs return data in JSON format (JavaScript Object Notation). It looks like objects and arrays in JavaScript and is easy to work with.

Below is a simple example that fetches a list of users from a free test API and shows them on the page as cards.

// JavaScript: Fetch users and display them
function loadUsers() {
  const output = document.getElementById("apiOutput");
  output.textContent = "Loading users...";

  fetch("https://jsonplaceholder.typicode.com/users")
    .then(response => {
      if (!response.ok) {
        throw new Error("Network response was not ok. Status: " + response.status);
      }
      return response.json();
    })
    .then(users => {
      output.textContent = "";
      users.forEach(user => {
        const div = document.createElement("div");
        div.className = "user-card";
        div.innerHTML = `
          
${user.name}

          
Email: ${user.email}

          
City: ${user.address.city}

        `;
        output.appendChild(div);
      });
    })
    .catch(error => {
      output.textContent = "Error fetching users: " + error.message;
    });
}
Click "Load sample users" to fetch data from a public test API.

4Ô∏è‚É£ Understanding promises (.then and .catch)

Fetch returns a promise, which resolves when the response arrives or rejects on network errors.

You chain .then() to process the response and .catch() to handle errors.

fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => response.json())  // convert to JSON
  .then(post => {
    console.log("Title:", post.title);
  })
  .catch(error => {
    console.error("Error:", error);
  });

5Ô∏è‚É£ Basic error handling with Fetch

Fetch does not throw errors for HTTP status codes by itself, so you should check response.ok and throw an error if it is false.

This lets you show friendly messages instead of silent failures.

fetch("https://jsonplaceholder.typicode.com/unknown-endpoint")
  .then(response => {
    if (!response.ok) {
      throw new Error("HTTP error! Status: " + response.status);
    }
    return response.json();
  })
  .then(data => {
    console.log("Data:", data);
  })
  .catch(error => {
    console.error("Something went wrong:", error);
  });

// JavaScript: Load a random quote
function loadQuote() {
  const box = document.getElementById("quoteBox");

    .then(response => {

        throw new Error("Status: " + response.status);

      return response.json();
    })
    .then(data => {
      box.textContent = `"${data.content}" ‚Äî ${data.author}`;
    })
    .catch(error => {
      box.textContent = "Could not load quote: " + error.message;
    });
}

7Ô∏è‚É£ GET vs POST (simple overview)

GET is used to read data, POST is used to send or create data on the server.

// Example of a POST request (demo)
fetch("https://jsonplaceholder.typicode.com/posts", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    title: "My first API post",
    body: "This is just a test.",
    userId: 1
  })
})
  .then(response => response.json())
  .then(data => {
    console.log("Created post:", data);
  })
  .catch(error => {
    console.error("Error creating post:", error);
  });
