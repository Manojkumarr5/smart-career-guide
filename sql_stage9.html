Step 9: Work with Views and Indexes

In this stage you learn what a view is in SQL and how it can simplify repeated queries, plus a basic idea of what an index does.

Views act like saved queries that you can treat as virtual tables, while indexes help the database search faster on specific columns.


1Ô∏è‚É£ What is a view?

A view is a virtual table based on the result of a SELECT query; it does not store its own data, but reads from underlying tables whenever you query it.[web:236][web:239]

  • Views can hide complex joins and filters behind a simple name.
  • They are useful for reports, security (showing only some columns or rows), and reuse.

2Ô∏è‚É£ Creating a simple view

The basic pattern for making a view is: CREATE VIEW view_name AS SELECT .... 

-- A view that shows basic student info
CREATE VIEW student_basic AS
SELECT id, name, branch
FROM students;

-- Use the view like a table
SELECT * FROM student_basic;

Whenever you SELECT from student_basic, the database actually runs the underlying SELECT query on students.

3Ô∏è‚É£ View with filter and join

Views can include WHERE, JOIN, GROUP BY, and other clauses so that reports are always generated with the same logic. 

-- CSE students only
CREATE VIEW cse_students AS
SELECT id, name, cgpa
FROM students
WHERE branch = 'CSE';

-- Students with their marks (join view)
CREATE VIEW student_marks AS
SELECT s.id,
       s.name,
       s.branch,
       m.subject,
       m.score
FROM students AS s
INNER JOIN marks AS m
    ON s.id = m.student_id;

-- Use these views
SELECT * FROM cse_students;
SELECT * FROM student_marks;

If underlying data changes, such as new students or marks, the view results update automatically because they are based on live tables.

4Ô∏è‚É£ Updating or dropping a view (concept)

You can change a view by dropping and recreating it, or by using ALTER VIEW if your database supports that syntax. 

-- remove a view when no longer needed
DROP VIEW cse_students;

-- pattern for ALTER VIEW (syntax may differ by database)
-- ALTER VIEW student_basic AS
-- SELECT id, name, branch, cgpa
-- FROM students;

In many beginner projects, dropping and recreating a view is enough for simple changes.

5Ô∏è‚É£ Indexes ‚Äì very basic idea

A database index is a special data structure that improves the speed of reading rows by allowing faster lookups on indexed columns.

  • Indexes are commonly created on columns used in WHERE or JOIN conditions, such as student_id or emails.
  • Indexes speed up reads but add write overhead because they must be updated on INSERT, UPDATE, and DELETE.
-- example pattern (exact syntax may differ)
CREATE INDEX idx_students_branch
ON students(branch);

CREATE INDEX idx_marks_student_id
ON marks(student_id);

Do not worry about advanced index tuning yet; just remember their purpose: faster lookups on frequently searched columns at the cost of extra storage and some write overhead.
