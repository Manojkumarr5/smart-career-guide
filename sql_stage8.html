Step 8: Write Subqueries and Nested Queries

In this stage you learn how to write subqueries (queries inside another query) to answer slightly more complex questions, plus a very small introduction to CTEs (WITH).

Subqueries are useful when one query needs the result of another query, such as ‚Äústudents with above‚Äëaverage CGPA‚Äù.


1Ô∏è‚É£ What is a subquery?

A subquery is a query nested inside another SQL statement, often in the WHERE, FROM, or SELECT clause.

  • The inner query runs first and produces a result set or single value.[web:226]
  • The outer query then uses that result to filter or compute its final output.

2Ô∏è‚É£ Subquery in WHERE ‚Äì compare with a single value

A common pattern is to use a subquery in WHERE to compare against an aggregate such as a maximum or average. 

-- students with cgpa equal to the maximum cgpa
SELECT name, branch, cgpa
FROM students
WHERE cgpa = (
    SELECT MAX(cgpa)
    FROM students
);

-- students with cgpa above overall average
SELECT name, branch, cgpa
FROM students
WHERE cgpa > (
    SELECT AVG(cgpa)
    FROM students
);

Here the inner query returns a single value (MAX or AVG), and the outer query compares each row‚Äôs cgpa with that value.

3Ô∏è‚É£ Subquery with IN ‚Äì filter by a list

When the subquery returns multiple values, you typically use operators like IN or NOT IN to filter rows based on that list. 

-- students who have at least one marks record
SELECT name
FROM students
WHERE id IN (
    SELECT DISTINCT student_id
    FROM marks
);

-- students who currently have no marks
SELECT name
FROM students
WHERE id NOT IN (
    SELECT DISTINCT student_id
    FROM marks
);

The inner query on marks returns a list of student_id values, and the outer query keeps only matching or non‚Äëmatching students.

4Ô∏è‚É£ Subquery in FROM ‚Äì derived table

A subquery in the FROM clause acts like a temporary table whose result you can query again, which helps break complex logic into clear steps. 

-- first compute average score per student in a subquery,
-- then select only those with avg_score >= 80
SELECT t.student_id,
       t.avg_score
FROM (
    SELECT student_id,
           AVG(score) AS avg_score
    FROM marks
    GROUP BY student_id
) AS t
WHERE t.avg_score >= 80;

Many people find this easier to read: first imagine the inner query‚Äôs result as a table, then filter it in the outer query.

5Ô∏è‚É£ Very short intro to CTE (WITH)

A Common Table Expression (CTE) is like a named subquery defined with WITH, which can make complex queries more readable and reusable. 

-- same idea as the previous example but with a CTE
WITH avg_marks AS (
    SELECT student_id,
           AVG(score) AS avg_score
    FROM marks
    GROUP BY student_id
)
SELECT student_id,
       avg_score
FROM avg_marks
WHERE avg_score >= 80;

CTEs are widely used in modern SQL to improve readability and can be referenced multiple times within the same query.
