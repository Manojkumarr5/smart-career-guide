Step 2: Understand Syntax, Loops, and Functions

In this stage you move from just ‚Äúrunning Python‚Äù to actually writing small programs. You will understand basic Python syntax rules, learn how loops repeat work for you, and how functions help you reuse code.

These skills are the foundation for projects, data work, and automation later in the Python track.


1Ô∏è‚É£ Python syntax ‚Äì how code is written

Syntax is the set of rules for how Python code must be written so that the interpreter understands it. If you break these rules, you get a SyntaxError.

  • Python uses indentation (spaces) to show blocks of code instead of curly braces.
  • Colons : are used after statements like if, for, while, and def.
  • One statement per line keeps code simple and readable.
  • Use # to write comments that explain your logic.
# Correct indentation example
if 5 > 3:
    print("Yes, 5 is greater than 3")


# Wrong (will cause indentation error)
if 5 > 3:
print("This will break")

Always use the same number of spaces for each indentation level (commonly 4 spaces). Mixing tabs and spaces can cause invisible errors.

2Ô∏è‚É£ for loop ‚Äì repeat over a sequence

A for loop lets you repeat a block of code for each item in a sequence like a list, string, or a range of numbers.

Basic pattern:

# Loop from 1 to 5
for i in range(1, 6):
    print("Number:", i)

Looping over a list:

fruits = ["apple", "banana", "mango"]

for fruit in fruits:
    print("I like", fruit)

The variable after for (like i or fruit) takes each value from the sequence one by one.

3Ô∏è‚É£ while loop ‚Äì repeat while condition is true

A while loop repeats as long as a condition stays true. It is useful when you do not know the exact number of repetitions in advance. 

count = 1

while count <= 5:
    print("Count:", count)
    count = count + 1   # update the variable

print("Loop finished")

Always make sure something inside the while loop changes the condition, or the loop can run forever.

4Ô∏è‚É£ Loop control ‚Äì break and continue

Two important keywords control loops:

  • break ‚Äì stop the loop completely.
  • continue ‚Äì skip the current step and go to the next one.
# Stop at 5
for i in range(1, 11):
    if i == 5:
        break
    print(i)
    
# Skip 5
for i in range(1, 11):
    if i == 5:
        continue
    print(i)

5Ô∏è‚É£ Functions ‚Äì reuse your code

Functions let you group statements together and give that group a name, so you can call it many times instead of repeating the same code.

Basic structure:

def greet(name):
    print("Hello,", name)

# calling the function
greet("Rahul")
greet("Aisha")
  • def starts a function definition.
  • name is a parameter (input for the function).
  • You can return a value to the caller using the return keyword.
def add(a, b):
    result = a + b
    return result

total = add(3, 4)
print("Total is:", total)

6Ô∏è‚É£ Putting it all together ‚Äì small program

Here is a small example mixing functions, loops, and conditions to show a simple marks report. 

def pass_or_fail(mark):
    if mark >= 35:
        return "Pass"
    else:
        return "Fail"

students = [("Asha", 40), ("Ravi", 30), ("Kiran", 90)]

for name, mark in students:
    status = pass_or_fail(mark)
    print(name, ":", mark, "-", status)

This structure (data list + loop + function) is very common in projects like result processing, bill calculation, and simple automation.
