Step 7: Use GROUP BY, HAVING, and Aggregation Functions

In this stage you learn how to summarize data using aggregate functions like COUNT, SUM, AVG, MIN, and MAX, together with GROUP BY and HAVING.

These tools let you answer questions such as ‚Äúaverage CGPA by branch‚Äù or ‚Äúhow many students scored above a certain value‚Äù.


1Ô∏è‚É£ Aggregate functions basics

Aggregate functions take many rows and return a single value such as a count, sum, or average. 

-- count all students
SELECT COUNT(*) AS total_students
FROM students;

-- average cgpa of all students
SELECT AVG(cgpa) AS avg_cgpa
FROM students;

-- minimum and maximum cgpa
SELECT MIN(cgpa) AS min_cgpa,
       MAX(cgpa) AS max_cgpa
FROM students;

COUNT(*) counts all rows, while COUNT(column) counts only rows where that column is not NULL.

2Ô∏è‚É£ GROUP BY ‚Äì summarize by category

GROUP BY groups rows that share the same values in one or more columns and then applies aggregates per group. 

-- average cgpa by branch
SELECT branch,
       AVG(cgpa) AS avg_cgpa
FROM students
GROUP BY branch;

-- number of students per branch
SELECT branch,
       COUNT(*) AS students_in_branch
FROM students
GROUP BY branch;

Every column in the SELECT list must be either inside an aggregate function or listed in the GROUP BY clause.

3Ô∏è‚É£ GROUP BY with multiple columns

You can group by more than one column, for example branch together with year_of_study or gender. 

-- example table students_extended with (branch, year_of_study)
-- count students per branch and year
SELECT branch,
       year_of_study,
       COUNT(*) AS total_students
FROM students_extended
GROUP BY branch, year_of_study
ORDER BY branch, year_of_study;

The GROUP BY columns define the ‚Äúlevel‚Äù of summarization that appears in the result.

4Ô∏è‚É£ HAVING ‚Äì filter after grouping

HAVING filters groups after aggregation, similar to how WHERE filters individual rows before grouping. 

-- branches where average cgpa is at least 8.0
SELECT branch,
       AVG(cgpa) AS avg_cgpa
FROM students
GROUP BY branch
HAVING AVG(cgpa) >= 8.0;

-- branches with more than 5 students
SELECT branch,
       COUNT(*) AS total_students
FROM students
GROUP BY branch
HAVING COUNT(*) > 5;

Use WHERE to filter rows before grouping and HAVING to filter the aggregated groups, especially when using aggregate functions in conditions.

5Ô∏è‚É£ Combined example: WHERE + GROUP BY + HAVING

In real scenarios you often filter rows, group them, filter groups, and finally sort the summary in one query. 

-- average cgpa by branch for only CSE and ECE students,
-- but show only branches where avg_cgpa >= 8.0, sorted by avg_cgpa
SELECT branch,
       AVG(cgpa) AS avg_cgpa
FROM students
WHERE branch IN ('CSE', 'ECE')
GROUP BY branch
HAVING AVG(cgpa) >= 8.0
ORDER BY avg_cgpa DESC;

A useful mental execution order is: WHERE (filter rows) ‚Üí GROUP BY (group rows) ‚Üí HAVING (filter groups) ‚Üí ORDER BY (sort results).
