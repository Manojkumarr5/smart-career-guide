Step 4: Work with SELECT, WHERE, and ORDER BY

In this stage you learn how to select data from tables, filter rows using WHERE, and sort results using ORDER BY; these clauses form the backbone of most queries.

All examples use the students table you created earlier, with columns such as id, name, branch, and cgpa.


1Ô∏è‚É£ SELECT ‚Äì choose which columns to see

SELECT defines which columns you want in the result, and FROM tells SQL which table to read. 

-- all columns, all rows
SELECT *
FROM students;

-- only specific columns
SELECT name, branch, cgpa
FROM students;

Start every query by asking: ‚ÄúWhich table am I reading from, and which columns do I need?‚Äù.

2Ô∏è‚É£ WHERE ‚Äì filter rows by condition

WHERE restricts the rows returned based on a condition such as a specific branch or a minimum CGPA. 

-- students from CSE only
SELECT name, branch, cgpa
FROM students
WHERE branch = 'CSE';

-- students with cgpa greater than or equal to 8.0
SELECT name, cgpa
FROM students
WHERE cgpa >= 8.0;

Comparison operators like =, <, >, <=, >=, and <> (or != in some systems) are used inside WHERE to express conditions.

3Ô∏è‚É£ WHERE with AND / OR

You can combine multiple conditions using logical operators AND and OR. 

-- CSE students with cgpa above 8.0
SELECT name, branch, cgpa
FROM students
WHERE branch = 'CSE' AND cgpa > 8.0;

-- students from CSE or ECE
SELECT name, branch
FROM students
WHERE branch = 'CSE' OR branch = 'ECE';

Use parentheses when mixing AND and OR to control the order of evaluation and make the logic clear.

4Ô∏è‚É£ ORDER BY ‚Äì sort results

ORDER BY sorts the result set by one or more columns; sorting is ascending (ASC) by default, or descending (DESC) if specified. 

-- sort students by name (A to Z)
SELECT name, branch, cgpa
FROM students
ORDER BY name ASC;

-- sort students by cgpa (highest first)
SELECT name, branch, cgpa
FROM students
ORDER BY cgpa DESC;

ORDER BY is applied after filtering, so SQL first selects rows with WHERE and then sorts those rows.

5Ô∏è‚É£ Combine WHERE and ORDER BY

In real queries you almost always combine SELECT, FROM, WHERE, and ORDER BY into a single statement. 

-- CSE students only, sorted by cgpa descending
SELECT name, branch, cgpa
FROM students
WHERE branch = 'CSE'
ORDER BY cgpa DESC;

-- students with cgpa >= 8, sorted by name
SELECT name, branch, cgpa
FROM students
WHERE cgpa >= 8.0
ORDER BY name ASC;

A helpful mental template is: SELECT ... FROM ... WHERE ... ORDER BY ...; and you fill in each part.
