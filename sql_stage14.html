Step 14: Connect SQL with Python

In this stage you learn how to connect Python to a database, run SQL queries from Python, and fetch results for apps or data analysis.

You start with SQLite, which needs no separate server, and then see how to connect to MySQL or other databases using drivers.


1Ô∏è‚É£ Using SQLite with Python (beginner-friendly)

Python‚Äôs built‚Äëin sqlite3 module lets you work with a local .db file without installing a separate database server. 

# file: basic_sqlite_example.py
import sqlite3

# 1) connect to (or create) a SQLite database file
conn = sqlite3.connect("college.db")  # will create if not existing
cursor = conn.cursor()

# 2) create a table (if it doesn't exist)
cursor.execute("""
CREATE TABLE IF NOT EXISTS students (
    id      INTEGER PRIMARY KEY,
    name    TEXT,
    branch  TEXT,
    cgpa    REAL
)
""")

# 3) insert some sample data
cursor.execute(
    "INSERT INTO students (name, branch, cgpa) VALUES (?, ?, ?)",
    ("Asha", "CSE", 8.5)
)
cursor.execute(
    "INSERT INTO students (name, branch, cgpa) VALUES (?, ?, ?)",
    ("Ravi", "ECE", 7.8)
)

conn.commit()  # save changes

# 4) run a SELECT query and fetch results
cursor.execute("SELECT id, name, branch, cgpa FROM students")
rows = cursor.fetchall()

for row in rows:
    print(row)

# 5) close the connection
cursor.close()
conn.close()

The ? placeholders are parameter markers that safely bind values and help prevent SQL injection.

2Ô∏è‚É£ Querying with pandas for analysis

For analysis, you can load SQL query results into a pandas DataFrame and then use Python for further processing. 

import sqlite3
import pandas as pd

conn = sqlite3.connect("college.db")

query = """
SELECT branch,
       AVG(cgpa) AS avg_cgpa,
       COUNT(*)  AS students_count
FROM students
GROUP BY branch
"""

df = pd.read_sql_query(query, conn)
print(df)

conn.close()

This pattern of using SQL to select data and pandas to analyze it is very common in data science workflows.

3Ô∏è‚É£ Connecting Python to MySQL (driver-based)

For MySQL you usually install a driver such as mysql-connector-python and then connect with host, user, password, and database settings. 

# install once:
# pip install mysql-connector-python

import mysql.connector

connection = mysql.connector.connect(
    host="localhost",
    user="your_username",
    password="your_password",
    database="college_db"
)

cursor = connection.cursor()

cursor.execute("SELECT id, name, branch, cgpa FROM students")
rows = cursor.fetchall()

for row in rows:
    print(row)

cursor.close()
connection.close()

Other databases such as PostgreSQL and SQL Server use similar patterns with their own drivers, for example psycopg2 or pyodbc.

4Ô∏è‚É£ Best practices: parameters, errors, and context managers

Use parameterized queries for user input and handle errors cleanly so your code is safe and robust. 

import sqlite3

try:
    conn = sqlite3.connect("college.db")
    cursor = conn.cursor()

    name   = "Meena"
    branch = "ME"
    cgpa   = 9.1

    cursor.execute(
        "INSERT INTO students (name, branch, cgpa) VALUES (?, ?, ?)",
        (name, branch, cgpa)
    )

    conn.commit()
except Exception as e:
    print("Error:", e)
    conn.rollback()
finally:
    cursor.close()
    conn.close()

Many programs also wrap connections in with blocks so resources are closed automatically even if an error occurs.
